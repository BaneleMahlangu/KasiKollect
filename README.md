# KasiKollect
# Waste Management Service Reporting System

## Project Overview
This project is a Django-based web application designed to improve community service delivery in Pretoria, Gauteng, specifically focusing on waste management. The system allows residents to report waste-related issues such as missed collections, illegal dumping, and overflowing bins, and track the status of their complaints.



##  Objectives
- Provide a simple and intuitive platform for reporting waste issues
- Improve communication between residents and the municipality
- Enable users to track the status of their complaints
- Address usability issues identified in existing systems (e.g., navigation, feedback, visibility)


## Features
- Report waste-related issues (missed collection, illegal dumping, overflowing bins)
- Step-by-step guided reporting process
- Complaint status tracking (Received → Assigned → In Progress → Resolved)
- User-friendly interface focused on simplicity and clarity
- Notification-ready system for updates



## Conceptual Design
The system is based on a **step-by-step interaction model** combined with a **dashboard metaphor**, allowing users to:
1. Select issue type  
2. Provide location details  
3. Submit report  
4. Track progress  

The design emphasizes:
- Visibility
- Feedback
- Consistency
- Learnability
- Efficiency


## 🏗️ Tech Stack
- **Backend:** Django (Python)
- **Frontend:** HTML, CSS (Bootstrap optional)
- **Database:** SQLite (default Django database)
- **Version Control:** Git & GitHub
